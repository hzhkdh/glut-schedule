# v0.20.0 Android → 微信端迁移说明

> 多学期课表的完整排障结论与刷新状态机同时记录在
> [v0.20.0-教务多学期调研.md](./v0.20.0-教务多学期调研.md) 的“成功排障经验”章节。微信端实现前必须逐项核对，不能复用旧版“无参数当前课表”刷新逻辑。

## 多学期数据契约

- 两端统一使用学期键 `<campus>:<portalYear>:<spring|autumn>`，标签统一为 `2026·春/秋`。
- 学期项保留门户 `year`/`term` 原值；桂林秋季为 `2`，南宁秋季为 `3`。
- 首次导入只返回从入学到最新可用学期的目录，当前学期自动导入，历史课表按需下载并离线缓存。
- 下一学期已有课程时必须出现在目录中，例如 2024 秋入学且 2026 秋已有课程时返回 5 项：2026秋、2026春、2025秋、2025春、2024秋。
- 历史选择只影响课表和学期总览；失败不得覆盖已有缓存。

Worker 保留协议 2、`loginAndGetSchedule` 及已有字段，并通过 `semesterCatalog`、`semesterSchedule` 能力提供目录和指定学期课表。服务端必须校验学期属于门户实际选项和用户入学范围。

### 微信端强制避坑项

- 当前与历史学期全部以【周次课表】逐周数据为唯一排课来源；个人课表仅负责初始化目标学期会话并补充教师等元数据，不得生成或覆盖 occurrence。
- 教务周次接口在同一 `JSESSIONID` 下共享“当前周次”可变状态。实测 5 路并发会让响应下拉框保留各自请求周次、课程表体却被覆盖成同一个随机周；微信端严禁在同一会话内并发请求不同周次。
- 每个学期必须严格执行“带 year/term 请求 `currcourse.jsdo` → GET 周次课表落地页 → 按周次升序逐页请求并完成校验 → 请求调课页”。全链路复用同一 Cookie 与 Referer，前一周完成后才能发起下一周；禁止跳过个人课表初始化或落地页直接 POST。
- 所有页面先暂存在内存，全部成功并校验后才按 semester id 原子写入；不要把部分成功结果落库。
- 将 `whichWeek` 下拉列表最大值持久化为学期 `portalMaxWeek`。历史学期用它决定页数；最新学期有权威校历时仍以校历为准。
- 每周响应必须同时校验学期标签和 `selectedWeek`；课程日期必须与星期一致，同页非空课程必须属于同一个自然周。首个非空周次推导出的学期周一须与后续所有非空周次一致。错位、跨周、认证过期或中途失败时不得覆盖旧缓存。
- 合法空周允许存在且不单独推导学期周一。教学楼、教室、开始时间与结束时间均不是课程有效性的必填项；中文特殊场地、体育、实习、线上或待定地点不得被过滤。
- 不要按固定下标读取周次表：真实页面存在额外“课节名称”列和外层嵌套布局表，必须按最内层表头的列名映射日期、课程名、星期、节次、教学楼和教室。
- 至少解析出一个星期与节次均有效的 occurrence 才能标记缓存成功；“有课程名但 occurrence 为 0”必须失败并允许重试。
- 周次语法按逗号分段后分别应用 `单/双`，`6-14双,16-18` 必须展开为 6/8/10/12/14/16/17/18。
- 调课记录唯一键必须包含原/新周次、星期、节次、教室；概览按原周次归组，纯补课才按补课周次归组。
- 请求参数必须使用门户 option 的原始 value，不能把显示年份直接当 `year`，也不能假设桂林、南宁秋季 `term` 相同。
- 解析节次必须兼容 `第1、2节`；fixture 至少保留一条中文顿号样本。
- 响应中可识别的选中学期与请求不一致时，整次请求失败并保留原缓存。
- 课程为零、结构未知、认证过期或网络失败时，不得先清空缓存；成功解析后再按 semester id 原子替换。
- 课表 Pager、当前周和缓存 URL 均按 semester id 隔离，切换学期时先切数据源、后设置周次。
- 区分“教务默认当前学期”和“最新已有课学期”；刷新后者时仍必须显式传入后者的 year/term。
- 状态颜色使用语义：缓存/下载成功为绿色，处理中为中性色，失败才为红色。

### 上课时间跨端契约

- 上课时间是用户按校区保存的显示配置，不参与 occurrence 解析；星期、周次与节次始终来自周次课表。
- 桂林模板为 14 行：第1～4节、午1、午2、第5～12节，内部编号 1～14；南宁模板为第1～11节，不含午1、午2。
- 桂林与南宁必须使用独立持久化键。解码失败、行数或内部编号不符、时间格式无效、开始不早于结束、相邻课时重叠时，只回退该校区默认值，不能污染另一校区。
- 设置二级页使用本地草稿；未保存返回不得改变生效配置。“恢复默认时间”先修改草稿，仍需用户点击保存。
- 有效配置必须统一用于当前和历史学期首页时间轴、课表详情、小组件课程起止时间及其刷新时点；微信端对应页面和组件也必须共用同一份校区配置，避免各自复制默认值后产生漂移。

## 校园信息同步

- 仍只保留一个“校园地图”标签，内部纵向展示雁山与屏风两张地图，不显示额外标题。
- 雁山类型保持 `/campus-image/campus-map`。
- 新增 `/campus-image/campus-map-pingfeng`，源图为 `https://img.999314.xyz/file/1784712704172_屏风校区地图.png`。
- 两张地图放在同一个 `movable-view` 中，共用缩放和平移。
- 已加载的各标签图片节点保持挂载，切换标签不重新请求；手动刷新仅替换当前标签请求。
- Worker Cache API 继续提供失败回退，成功图片响应使用 30 天缓存。

## UI 与公告

- 抽屉宽度从 60% 调整为屏幕的 75%，不再使用会限制实际宽度的 `max-width`。
- 公告保持原有单页纯文本列表，不增加详情路由或富文本渲染；长内容在页面内正常滚动。

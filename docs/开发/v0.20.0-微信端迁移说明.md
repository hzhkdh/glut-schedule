# v0.20.0 Android → 微信端迁移说明

> 状态：Android 先行。只有 Android 经用户验收后，才在 `scheduleApp_weixin` 实施本迁移；当前文档是同步契约，不代表 Worker 或小程序已经部署。

## 数据与缓存对齐

- 两端统一使用学期键 `<campus>:<portalYear>:<spring|autumn>`。
- 学期项保留门户 `year`/`term` 原值，桂林秋季为 `2`、南宁秋季为 `3`。
- 单账号模型不变，不增加账号切换或多账号缓存分区。
- 当前学期自动导入，历史学期按需下载；冷启动查看当前学期，历史缓存永久保留且只读。
- 历史选择只影响课表和学期总览；其他教务菜单不切换数据上下文。

## 后端增量契约

保留协议 2、`loginAndGetSchedule` 以及所有已发布字段和行为。Worker 与兼容云函数同步追加：

- capabilities：`semesterCatalog`、`semesterSchedule`。
- action `getSemesterCatalog`：复用现有 `campus`、`academicCookie`、账号密码及南宁验证码认证包络；返回刷新后的 Cookie、入学信息、`currentSemesterKey` 和 `semesters[]`。
- `semesters[]` 项：`key`、`portalYear`、`portalYearId`、`portalTermId`、`season`、`label`、`isCurrent`。
- action `getSemesterSchedule`：接收目录返回的 `semesterKey`。后端必须校验它属于门户实际选项和有效入学范围，再返回 `semester`、`html`、可选 `nanningAdjustmentHtml` 及刷新后的 Cookie。

新增字段只能追加，不能删除、改名或改变旧 action 的含义。后端先兼容、前端后启用；本任务验收前不部署 Worker、不上传小程序。

## 公告 Markdown

- `notices-mp.json` 字段结构保持不变，仍使用 `content`、图片字段和 `files`。
- 使用 Towxml 最小构建支持标题、粗斜体、删除线、列表、引用、分隔线、行内/块代码和 `http/https` 链接。
- 不解析内联 HTML，不启用 Markdown 图片；图片和附件继续走独立字段。
- 公告列表显示去格式摘要，详情页渲染 Markdown；小程序版本升级仍由微信平台管理，不新增自制更新检查器。

## 验收矩阵

- 旧线上客户端继续完成登录、当前课表导入和刷新。
- 新客户端分别验证桂林、南宁目录发现、秋季 term 映射、历史导入、空学期、失败重试、离线切换和冷启动回当前。
- `node --test` 覆盖旧/新接口契约、解析器、存储与公告页面；微信开发者工具模拟器和真机复核触摸、滚动、链接与附件。
- 所有 fixture 使用合成数据，不提交真实账号、Cookie、验证码或个人 HTML。

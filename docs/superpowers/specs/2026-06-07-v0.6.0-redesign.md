## 一、导航布局重构

### 1.1 现状

`MainActivity.kt` 使用 `sealed class MainScreen` + `mutableStateOf` + `when` 切换屏幕：

```kotlin
sealed class MainScreen {
    data object Schedule : MainScreen()
    data object AcademicImport : MainScreen()
    data object ExamSchedule : MainScreen()
}
```

### 1.2 目标

改用 Material 3 `ModalNavigationDrawer`，6 个菜单项：

| 菜单项 | 图标 | 说明 |
|--------|------|------|
| 课程表 | CalendarMonth | 现有 ScheduleScreen |
| 考试成绩 | Assessment | **新增** ScoreScreen |
| 考试安排 | EventNote | 现有 ExamScreen |
| 导入课表 | Download | **重构** DirectLoginScreen（替代 AcademicImportScreen） |
| 设置 | Settings | 抽离自 ScheduleScreen 弹窗 |
| 关于 | Info | 抽离自 ScheduleScreen 弹窗，扩展内容 |

### 1.3 实现方式

**不引入 Jetpack Navigation 组件**，保持简洁的手动切换：

```kotlin
enum class DrawerItem(val title: String, val icon: ImageVector) {
    Schedule("课程表", Icons.Default.CalendarMonth),
    Score("考试成绩", Icons.Default.Assessment),
    Exam("考试安排", Icons.Default.EventNote),
    Import("导入课表", Icons.Default.Download),
    Settings("设置", Icons.Default.Settings),
    About("关于", Icons.Default.Info),
}
```

- `ModalNavigationDrawer` + `rememberDrawerState` 包裹 Scaffold
- `drawerContent` 包含 Logo 头部 + 菜单列表
- 内容区 `when(selectedItem)` 切换页面
- TopAppBar 标题随选中页面变化，左侧汉堡菜单按钮
- 移除 WebView 相关代码：`AcademicImportScreen`、`AcademicImportViewModel`

### 1.4 待移除的组件

- `AcademicImportViewModel.kt` — WebView 导入 ViewModel

## 二、直接登录导入课表

### 2.1 现状

现有 `AcademicLoginService` 已实现直接 POST 登录 `j_acegi_security_check`，支持：
- 用户名 + 密码 + 验证码
- 保存 Cookie 到 `AcademicSessionStore`
- 登录状态枚举：Success / MissingCredentials / InvalidCredentials / CaptchaOrInteractiveLoginRequired / NetworkError

但导入流程仍强制走 WebView。

### 2.2 目标

新建 `DirectLoginScreen` 替代 WebView 导入：

```
┌─────────────────────────────────┐
│  ← 导入课表                      │
├─────────────────────────────────┤
│                                 │
│  学号  [____________________]   │
│  密码  [____________________]   │
│                                 │
│  ┌──────────┐  ┌──────────┐    │
│  │ 验证码图片 │  │ [刷新]    │    │
│  └──────────┘  └──────────┘    │
│  验证码 [________]              │
│                                 │
│  [ 记住密码 ☐ ]                  │
│                                 │
│  [      登 录 并 导 入       ]   │
│                                 │
└─────────────────────────────────┘
```

**流程**：
1. 进入页面 → 加载验证码图片（`getCaptcha.do`），自动填充已保存的学号/密码
2. 用户填写验证码 → 点击登录
3. 调用 `AcademicLoginService.login()` → 成功获取 Cookie
4. 调用 `ApiProbeService.probeAllEndpoints()` 抓取课表 → 解析 → 存入 Room
5. 同时抓取考试安排、考试成绩 → 一并存入
6. 显示导入结果（课程数、考试数、成绩数）

### 2.3 实现文件

- **新增** `DirectLoginScreen.kt` — 登录表单 UI
- **新增** `DirectLoginViewModel.kt` — 登录状态管理
- **复用** `AcademicLoginService` — HTTP 登录逻辑
- **复用** `AcademicLoginHttpClient` + `CapturingCookieJar` — 验证码图片获取

## 三、考试成绩查询

### 3.1 目标

新增考试成绩页面，支持按学期切换查看历史成绩。

### 3.2 教务接口

参考 GlutAssistant 和 glut 项目的实现：
- **URL**: `POST /academic/manager/score/studentOwnScore.do`
- **参数**: `year`（学年）、`term`（学期）、`para`（参数）
- **返回**: HTML 表格，包含课程名称、成绩、绩点等

### 3.3 数据模型

```kotlin
// 新增 Room Entity
@Entity(tableName = "score")
data class ScoreEntity(
    @PrimaryKey val id: String,
    val courseName: String,
    val score: String,
    val gpa: Double,
    val credit: Double,
    val year: String,
    val term: Int,
    val category: String,
    val examType: String
)
```

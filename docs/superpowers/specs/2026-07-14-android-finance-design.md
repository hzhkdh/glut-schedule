# Android 财务菜单设计

## 目标

在 Android App 中新增只读“财务”菜单，直接请求桂林理工大学财务官网，提供与微信端一致的财务查询能力。首版仅支持桂林校区；南宁校区保留入口并显示暂不支持。

发布版本为 `0.18.0`。本次只构建本地 Release APK，不自动发布 GitHub Release 或更新服务。

## 用户体验

### 导航与视觉

- 抽屉“成绩与服务”分组在“体测成绩”后新增“财务”。
- 页面采用已确认的 A1 方案：`概览 / 缴费 / 记录 / 学分` 四个主标签。
- 主绿色为 `#244F46`，辅助绿色为 `#397267`，金额强调色为 `#BD573E`，页面背景为 `#F5F1E9`。
- 顶栏保留抽屉按钮、标题和刷新按钮；刷新只作用于当前模块。
- 页面进入时优先显示本地缓存。切换标签或子模块只读取缓存，不自动发起网络请求。

### 模块映射

- 概览：欠费、应收、已缴、缓缴、待缴项目及费用摘要。
- 缴费：待缴项目、其他缴费、收费项目。
- 记录：交易记录、缴费明细、选课记录、电子票据。
- 学分：学分结算的多个动态表格区块。
- 电子票据记录可继续请求票据图片并预览。
- 需要分页的记录模块支持加载更多，分页结果只合并到对应模块。

### 登录与外部跳转

- 财务账号、密码、Cookie 与教务、体测数据完全隔离。
- 默认记住学号和财务平台密码，并使用 Android Keystore 支持的加密偏好存储。
- 会话失效后自动弹出已经填好账号密码的新验证码登录框；用户只需输入验证码。
- 登录成功后继续执行会话失效前被中断的刷新操作。
- 验证码错误只刷新验证码，不清空账号密码。
- “前往财务官网”和“重置密码”通过系统浏览器打开。
- App 内不执行实际支付。

## 架构

### 模块边界

- `data/model/FinanceModels.kt`：稳定的财务领域模型和模块枚举。
- `service/finance/FinanceApiService.kt`：官网 HTTP 请求、Cookie 合并、登录流程和模块查询。
- `service/finance/FinanceParser.kt`：把官网 JSON/动态表格转换为领域模型。
- `service/finance/FinanceStore.kt`：加密凭据与会话、按模块隔离的本地缓存和缓存时间。
- `ui/pages/FinanceViewModel.kt`：页面状态、当前模块刷新、登录恢复、分页和错误处理。
- `ui/pages/FinanceScreen.kt`：Compose 页面、登录弹窗、详情底部面板和票据预览。

UI 不读取官网原始响应，也不直接处理 Cookie。网络层不依赖 Compose 状态。

### 请求与会话

- 使用财务专用 `OkHttpClient` 和 Cookie 管理，直连 `https://cwjf.glut.edu.cn/`。
- 登录流程严格复现已经在微信端验证的官网顺序：创建会话、获取验证码、提交登录、初始化首页会话、查询模块。
- 请求参数、Referer、AJAX 头和分页字段与官网一致。
- 任何响应出现 `usernologin` 均归一化为会话过期。
- 概览可以组合必要的个人财务摘要与待缴数据，但普通模块刷新不得请求或返回其他模块。
- 不在日志中输出密码、Cookie、验证码或完整财务响应。

### 缓存

- 凭据和 Cookie 保存到独立的 `EncryptedSharedPreferences`。
- 归一化后的模块数据与保存时间存入独立普通私有缓存。
- 更新一个模块时只替换该模块缓存；其他模块和分页状态保持不变。
- 网络失败、官网超时或解析失败时保留旧缓存。
- 清除 App 数据时同步清除财务凭据、会话和全部模块缓存。

## 页面状态

- `FinanceUiState` 保存当前主标签、子模块、模块数据、缓存时间、分页、刷新状态、提示信息和登录弹窗状态。
- 同一时间只允许一个刷新或登录请求。
- 没有缓存时显示明确空状态和“点击右上角刷新”。
- 南宁校区显示占位说明，不创建财务网络请求。
- 学分表格区域支持双向滚动；表格横向拖动不触发抽屉手势。
- 电子票据、记录明细使用底部面板展示，避免创建大量导航目的地。

## 错误处理

- 网络、超时和格式错误转换成稳定的中文提示。
- 会话过期清除 Cookie，但保留账号、密码和缓存。
- 登录或验证码错误保留输入信息并刷新验证码。
- 解析到空数据时区分“暂无记录”和“官网响应异常”。
- 取消协程或页面退出不得把旧请求结果覆盖到新状态。
